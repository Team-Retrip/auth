📋 PRD 리뷰 & 업계 표준 반영 결과

✅ 결정 사항
1. 알림: 마스터 ON/OFF만 구현 (세부 토글 다음 배포)
2. 인증 잠금: 이름·생년월일 isVerified 체크
3. 소셜 비밀번호: API 차단 + UI 숨김
4. 탈퇴: Soft Delete 유지
5. 비밀번호 정책: 업계 표준 적용
6. MBTI 검증: 다음 배포
🔐 비밀번호 정책 (업계 표준)
• 최소 8자 ~ 최대 20자
• 영문 대소문자 / 숫자 / 특수문자 중 3가지 이상
• 허용 특수문자: !@#$%^&*()-_=+[]{}|;:,./
• @Pattern 정규식: ^(?=.*[a-zA-Z])(?=.*\d)(?=.*[!@#$%^&*]).{8,20}$
• 참고: 카카오/네이버 기준
🌐 소셜 로그인 비밀번호 (업계 표준)
소셜 가입 유저는 소셜 플랫폼에서 비밀번호 관리
→ 앱 내 비밀번호 없음 (null) → hasPassword() == false
→ API에서 hasPassword() == false 시 403 반환 ✅ 구현완료
→ 프론트에 hasPassword 필드 전달 → UI 숨김
참고: 카카오톡, 네이버, 토스 동일 정책
⚠️ 회원탈퇴 PRD 보완 제안
현 PRD: "모든 개인정보 즉시 파기"
현 코드: Soft Delete만 (isDeleted=true)
→ 법적 요구사항 충족 위해:
소셜 유저: CI/DI null 처리
이메일 유저: 재가입 불가 체크용 해시만 보존
개인식별정보(이름, 이메일원문)는 마스킹 권장
