13. 시나리오 검증
시나리오 A — 닉네임 자동 설정 검증
닉네임 없이 가입 시 GET /users/me 응답의 nickname 값이 name과 동일한지 확인
  1. nickname 미입력으로 회원가입
  2. GET /users/me 호출
  3. 응답의 nickname == name 확인 ✅
시나리오 B — hasPassword 분기 검증
현재 로그인된 계정 기준. 소셜 전용 계정(hasPassword=false)에서 비밀번호 관련 API 동작 확인.
  1. GET /api/users/profile → hasPassword 값 확인
  2. hasPassword=false면: POST /users/password → 200 성공
  3. hasPassword=false면: PATCH /users/password → 403 Member-007
  4. hasPassword=true면: POST /users/password → 409 Member-008
현재 저장된 토큰으로 실행됩니다. 소셜 로그인 후 토큰을 저장하세요.
시나리오 C — isVerified 잠금 검증
본인인증 완료 후 이름/생년월일 변경 시도 → 400 Member-006, 성별 변경은 가능 → 200
  1. 현재 isVerified 상태 확인 (GET /users/me)
  2. PUT /users {name: "변경시도"} → 400 Member-006 ✅
  3. PUT /users {gender: "F"} → 200 성공 ✅
본인인증(섹션 4)이 완료된 계정에서 실행하세요.
시나리오 D — 소셜 계정 연동 검증 (이메일 자동 링크)
동일 이메일로 다른 소셜 로그인 시 linkedProviders 배열에 항목이 추가되는지 확인
  1. 카카오로 로그인 (섹션 2)
  2. GET /api/users/profile → linkedProviders 배열 확인
  3. 같은 이메일로 네이버 로그인
  4. GET /api/users/profile → linkedProviders에 naver 추가 확인 ✅
현재 저장된 토큰으로 프로필을 조회해서 linkedProviders를 확인합니다.
시나리오 E — 소셜 유저 탈퇴 검증
비밀번호 없는 소셜 전용 계정에서 탈퇴 (body 없이) → 204 성공
  1. 소셜 로그인 후 토큰 저장 (섹션 2)
  2. GET /users/me → hasPassword=false 확인
  3. DELETE /users (password 없이) → 204 ✅
  4. 동일 이메일로 재가입 시도 → 400 Member-003 ✅
실행 시 실제로 탈퇴됩니다. 테스트 계정에서만 사용하세요.

15. 아이디 / 비밀번호 찾기
본인인증 경로: 상단 Config 바에 PortOne Store ID와 Channel Key가 입력돼 있어야 합니다.
이메일 경로: 서버에 MAIL_USERNAME / MAIL_PASSWORD 환경변수가 설정돼 있어야 합니다.

POST /auth/find-email  아이디 찾기 (본인인증)

인증 불필요. PortOne 본인인증 후 maskedEmail 반환. isNowVerified=true면 이번 요청에서 본인인증이 처음 연결된 것.
또는 impUid 직접 입력 (팝업 불가 시)
POST /auth/find-email
CI 미매칭: 본인인증 정보와 연결된 계정을 이름+생년월일로 탐색했습니다.
등록된 이메일로 OTP를 발송해 이메일 소유권을 확인합니다.
POST /auth/find-email/send-code
POST /auth/find-email/confirm

POST /auth/password-reset/by-email  비밀번호 찾기 (이메일)

인증 불필요. 입력한 이메일로 재설정 링크 발송. 소셜 전용 계정은 Auth-005 오류.
POST /auth/password-reset/by-email
메일함에서 링크의 ?token= 값 복사 후 아래에 붙여넣기
POST /auth/password-reset

POST /auth/password-reset/by-verification  비밀번호 찾기 (본인인증)

인증 불필요. PortOne 본인인증 후 재설정 토큰 발급. 소셜 전용 계정은 Auth-005 오류.
또는 impUid 직접 입력
POST /auth/password-reset/by-verification
CI 미매칭: 이름+생년월일로 계정을 탐색했습니다.
가입 시 사용한 이메일을 입력하면 OTP로 소유권을 확인합니다.
POST /auth/password-reset/by-verification/send-code
POST /auth/password-reset/by-verification/confirm
위에서 발급된 토큰이 자동 입력됩니다.
POST /auth/password-reset
